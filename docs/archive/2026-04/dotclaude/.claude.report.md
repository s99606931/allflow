## 6. 학습된 패턴 (메모리 적재)

### 6.1 정합성 5계층 일관 원칙

분업화 의도가 다음 5계층에서 동시 표현될 때 갭이 최소화된다:
1. 프로토콜 § (RACI/경계/핸드오프)
2. 에이전트 frontmatter + 본문 (책임표 + 경계 명시문)
3. CODEOWNERS (실 GitHub 위임)
4. 훅 (자동화 게이트)
5. 워크플로우 (CI/CD 강제)

한 계층만 빠져도 Contract 점수 -2~5%p 발생. (Contract 96.5% → 99% 차이가 G3·G4 한 줄 명시 누락에서 발생)

### 6.2 메타 vs 실코드 게이트 구분 패턴

av가 "메타 프로젝트" 단계에서 모든 게이트를 강제하면 false-fail. 해법:
- SKILL/규칙에 "활성화 조건: 실코드 추가 시" 단서 부착
- 현 단계 게이트와 미래 게이트 분리 명시
- 실코드 추가 PR이 들어오면 자동 활성화되는 잡 템플릿 미리 작성

### 6.3 Placeholder 자가 점검 패턴

OSS 공개 전 SECURITY.md / CODE_OF_CONDUCT.md / CODEOWNERS의 placeholder는 사람이 잊기 쉬움.
- `/av-oss-init --check-only` 마지막 단계에 grep 자동 점검
- 패턴: `{도메인}|{domain}|TODO|FIXME|example.com` + `@autovibe-{role}-leads`
- 발견 시 공개 차단 권고
